# Text

> **Source**: BMW Group Density Design System
> For the most up-to-date guidelines, visit:
> https://density.bmwgroup.net/v11/content/voice-and-tone

---

For messages and longer texts, use the general semantics and syntax for capitalization and punctuation.

## 1. Spelling

Content of BMW Group business applications refer to the official dictionary and spelling guide of the respective language for guidance regarding spelling:

- **English**: Oxford Dictionary
- **German**: Duden

### Capitalization

All interface element (labels, text) should use sentence case. Capitalize only the initial letter of the first word in the text and other words that require capitalization, such as proper nouns. Same guideline is applicable for German.

**Do:**

> "Driver's license number"

**Don't:**

> "Driver's License Number"

## 2. Punctuation

In general, punctuation marks refer to the Oxford English respectively German Duden rules. When it comes to UI writing follow those guides:

### Exclamation Mark

Use exclamation marks sparingly. Do not try to generate excitement with an exclamation mark (only use it if the user is actually experiencing something exciting). You can also use an exclamation mark for something cautionary, like "Stop!". To more accurately capture human expression, use an exclamation mark after just a few words, not after a long sentence.

Never use exclamation points in failure messages or alerts. When in doubt, avoid! ;-)

**Do:**

> "Congratulations! You created an account."

**Don't:**

> "Congratulations on creating an account!"

### Hyphen and Dash

A hyphen joins two or more words together while a dash separates words into parenthetical statements. Generally, hyphens are used to join two words or parts of words together while avoiding confusion or ambiguity. Hyphens are not separated by spaces.

Dashes can be used to separate words into parenthetical statements or to add comments in much the same way as you would use brackets. Using a dash is considered less formal then brackets. Dashes can be used to create emphasis in a sentence. A dash has a space on either side.

Example: "We might come to the party – you never know."

**Do:**

> up-to-date

**Don't:**

> first- time

## 3. Ampersands

Avoid using ampersands and use "and" instead. Only use them if they are part of a brand name.

**Do:**

> Ben & Jerry

**Don't:**

> Car & Motorbike

## 4. Parentheses

Do not use parentheses to indicate a possible plural of something, like "Material(s)." If a user can select one thing or multiple things, use the plural form.

**Do:**

> Materials

**Don't:**

> Material(s)

## 5. Personal Names

User information are following the structure `<First name> <Last name> | <Department> | <Role>`. Displaying the role is optional whereas the department is mandatory.

External employees are displayed with the commissioning department:

- **English**: On behalf of
- **German**: im Auftrag

**Do:**

> John Doe | OBO FG-123

**Don't:**

> Mary Doe | External support for FG-123

## 6. Addressing People and Gendering

The basic rule is that our written texts are gender-neutral. To the greatest possible extent, we avoid gender-specific language.

If the language you are writing in uses gender-specific forms of address, ensure that you use the correct version in accordance with the language rules. If the language you are writing in differentiates between formal and informal forms of address, ensure that you make the appropriate choice for the respective channel or context.

**Reference**: Please also refer to Gendersensible Sprache BMW Group (German only).

### Direct Address

Address the user directly as "you". Never gender "the user" with second person pronouns ("she"/"he").

**Do:**

> Your password

**Don't:**

> User password

### German Gender-Inclusive Language

To ensure that all users are addressed, the machine-readable Genderdoppelpunkt ":" is used in the German spelling.

**Do:**

> Anwender:innen

**Don't:**

> Anwender

### Gender-Neutral Language

Using gender-neutral language is even better. Only when it makes sense — please don't invent new words 😅.

**Do:**

> Liebe Mitarbeitende

**Don't:**

> Liebe Mitarbeiter und Mitarbeiterinnen
