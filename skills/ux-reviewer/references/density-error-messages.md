# Error Messages

> **Source**: BMW Group Density Design System
> For the most up-to-date guidelines, visit:
> https://density.bmwgroup.net/v11/content/voice-and-tone

---

Error messages can be disturbing. Make errors visible to the users, easy to understand, and helpful. Explain the users where the error happened, what happened, why it happened, what to do next.

## Don't Blame Users

Users should never feel like the error is their fault.

**Do:**

> "Material number is invalid."

**Don't:**

> "You entered the wrong material number."

## No Drama

Explain the situation clearly and how it can be resolved. Do not use overly dramatic or frightening words for simple errors. Do not use words like "bad, forbidden, fatal, expectation failed".

**Do:**

> "Material weight needs to have a positive value. Change the weight to be greater than 0 and try again."

**Don't:**

> "Fatal error occurred: It is bad to enter negative values. This input is forbidden."

## Include Instruction

Tell the users what is wrong. An error without an explanation can add to their frustration.

**Do:**

> "To send this request, extend the access permissions in user settings."

**Don't:**

> "The request cannot be sent due to user settings."

## Avoid Technical Terms

Translate all technical errors into human-readable messages.

**Do:**

> "Your session is outdated. To keep it active, refresh the page."

**Don't:**

> "500 - Internal Server Error"

## Speak Your Users Language

Use terms that are familiar to the target audience. If your product has several roles of users (e.g. developers and designers), it's better to write a separate message for each role. Additionally avoid abbreviations.

**Do:**

> "Please approve your shopping cart through creating a confirmation."

**Don't:**

> "Please create a LB to approve the SC."

## Give Users a Next Step

Users should never feel stuck. Provide information they need to continue with their task.

**Do:**

> "Your list reached the maximum number of material numbers. Remove one material number to continue."

**Don't:**

> "Your list reached the maximum number of items. You are not able to continue selecting more."

## Brief and Simple

Keep the explanation compact but specific. What went wrong and what to do next.

**Do:**

> "Only 1 material number can be saved. To continue, remove the other one."

**Don't:**

> "Your information cannot be saved. Our system is currently designed to change 1 material at a time. The system is unable to cope with more than 1 item at a time."

## Be Descriptive and Actionable

Rather than pointing out that there are 'X' errors, explain what can be done to resolve them. For an instruction use actionable words.

**Do:**

> "To save the material number, 2 changes need to be made:
>
> - Enter a material code.
> - Add a weight above 0."

**Don't:**

> "There are 2 errors with this material number:
>
> - Material code can't be blank.
> - Weight is not acceptable."
