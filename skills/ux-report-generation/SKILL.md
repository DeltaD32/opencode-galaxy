# Unified Report Generation

This skill defines the **single source of truth** for all HTML evaluation reports generated by DesignOps skills. Every skill that produces a report must load and follow this skill to guarantee visual and structural consistency.

## Skill Structure

```
.github/skills/ux-report-generation/
├── SKILL.md                    ← This file — orchestration & workflow
├── docs/
│   ├── report-contract.md      ← Required sections, Density tokens, rating/severity reference
│   └── screenshot-guidance.md  ← Screenshot capture, SSO handling, placeholder blocks
├── templates/
│   ├── report-shell.html       ← Full HTML structure with [PLACEHOLDER] markers
│   └── report-styles.css       ← All Density CSS tokens and component styles
├── scripts/
│   └── generate-pdf.mjs        ← Playwright PDF export script
└── tests/
    └── functional/
        ├── test_basic_report_generation.yaml        ← All 7 sections generated and written to disk
        ├── test_density_token_compliance.yaml        ← CSS uses Density tokens, no hardcoded hex
        ├── test_edge_case_no_screenshots.yaml        ← Placeholder blocks when capture fails
        └── test_error_handling_missing_context.yaml  ← No fabricated report without findings
```

## When to Use

Load this skill whenever a parent skill instructs you to generate a visual HTML report. Do not invent a separate report template — always follow the specification in this skill.

Before generating a report, read the supporting docs in order:

1. `docs/report-contract.md` — required sections, Density tokens, rating/severity scale
2. `docs/screenshot-guidance.md` — how to capture and embed screenshots

**File creation is fully automated — never ask the user to manually save files or run scripts.** When generating a report:

1. Read `templates/report-styles.css` to get the current source.
2. Use `templates/report-shell.html` as the structural blueprint.
3. Use `create_file` to write the completed HTML report (CSS inlined) to the correct folder.
4. Use `create_file` to copy `scripts/generate-pdf.mjs` into the same report folder, replacing `[REPORT_FILE_NAME]`.
5. Use `run_in_terminal` to execute `node generate-pdf.mjs` and produce the PDF.

> **Reports are self-contained single HTML files.** Inline all CSS inside `<style>`. Do not reference external files — reports must render without a server.

## Post-Generation Steps (Automated)

Execute all steps using tools — **never ask the user to perform them manually**.

> **Order is mandatory:** Screenshots must be saved to disk **before** the HTML file is written. The HTML embeds relative `src` paths — if images don't exist when the file is opened, they will appear broken.

1. **Capture and save screenshots** — for every finding where visual evidence adds value, use `run_playwright_code` to navigate to the relevant page state and save a PNG directly to disk:

   ```js
   // Use run_playwright_code — NOT screenshot_page
   await page.goto("https://...", { waitUntil: "networkidle", timeout: 15000 });
   await page.waitForTimeout(500); // allow animations to settle
   await page.screenshot({
     path: "/absolute/path/to/reports/<skill-folder>/screenshots/<finding-name>.png",
     fullPage: false,
   });
   ```

   > ⚠️ **Do NOT use `screenshot_page`** — it saves images to VS Code chat resource URIs (temporary, not accessible from disk). Only `run_playwright_code` with `page.screenshot({ path })` persists files that can be referenced in the HTML report.

   **If the browser session has ended before screenshots are captured:**
   Re-open the browser with `open_browser_page`, navigate to each URL that produced a finding, and re-capture. For SSO-protected applications, follow the non-headless / user-pause procedure in `docs/screenshot-guidance.md`.

   Save each screenshot to:

   ```
   reports/<skill-folder>/screenshots/<descriptive-name>.png
   ```

   See `docs/screenshot-guidance.md` for SSO handling and placeholder rules.

2. **Write the HTML report** — use `create_file` with the full absolute path:
   `reports/<skill-folder>/<app-name>-<type>-report.html`
   The report must be a single self-contained HTML file with all CSS inlined. Embed screenshots using relative paths (`screenshots/<file>.png`) inside `<div class="screenshot">` blocks.

3. **Write the PDF export script** — copy `scripts/generate-pdf.mjs` to the same report folder and replace `[REPORT_FILE_NAME]` with the actual file name (without `.html`).

4. **Generate the PDF** — use `run_in_terminal`:

   ```bash
   cd reports/<skill-folder> && node generate-pdf.mjs
   ```

   **This is mandatory.** Both `.html` and `.pdf` are required deliverables. If PDF generation fails, report the error explicitly and still deliver the HTML.

5. **Confirm outputs** with the paths to all deliverables:
   - `reports/<skill-folder>/<app-name>-<type>-report.html`
   - `reports/<skill-folder>/<app-name>-<type>-report.pdf`
   - `reports/<skill-folder>/screenshots/*.png` (list captured files)
